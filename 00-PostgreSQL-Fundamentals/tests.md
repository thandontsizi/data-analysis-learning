# PostgreSQL Fundamentals: Navigation Tests.
These tests verify my ability to access PostgreSQL, navigate psql, inspect metadata, and manage sessions. 
No SQL queries are required.

## Section 1: Accessing PostgreSQL.
1. You log into a Linux server that has PostgreSQL installed. Write the command to switch to the PostgreSQL system user.
2. From the PostgreSQL system user, open the psql interactive terminal.
3. Exit the psql terminal safely.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 2: Database Navigation.
1. You are inside psql and want to see all available databases. Write the command to list them.
2. You need to switch your session to a different databse. Write the command to connect to another database.
3. You want to confirm which database and PostgreSQL user you are currently connected as. Write the command to display connection information.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 3: Schema and Table Inspection.
1. After connecting to a database, you want to see which tables exist. Write the command to list the tables in the current schema.
2. You suspect tables might exist in other schemas. Write thr command to list all tables across all schemas.
3. You want to understand the structure of a table without querying its data. Write the command to display a table's structure.
4. You want to see all schemas defined in the databse. Write the command to list schemas.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 4: Help and Discovery.
1. You are new to psql and you want to explore available meta-commands. Write the command to display all psql commands.
2. You want to read SQL commands without executing anything. Write the command to open general SQL help.
3. You want help for a specific SQL command. Write the command to view help for a single command.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 5: Session Utilities.
1. A query output is difficult to read in the terminal. Write the command to toggle expanded output mode.
2. You want to review commands you've run during a session. Write the command to view command history.
3. A command is taking too long to respond. Write the keyboard shortcut to cancel it safely.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 6: Environment Awareness.
1. You want to verify which Linux user you are currently logged in as. Write the command to check this.
2. You want to verify which PostgreSQL user and database your psql session is using. Write the command to check connection details.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Section 7: Backup and Restore (Conceptual).
1. You are asked to backup a PostgreSQL database. Write the general command used for database backups.
2. You are asked to restore a PostgreSQL database from a backup file. Write the general command used for database recovery or restoration.
-------------------------------------------------------------------------------------------------------------------------------------------------

## Completion Criteria:
This test is complete when I can:
- Navigate PostgreSQL confidently without panic.
- Move between databases intentionally.
- Inspect metadata without querying data.
- Use help and session tools naturally.
- Explain PostgreSQL tooling clearly in interviews.
